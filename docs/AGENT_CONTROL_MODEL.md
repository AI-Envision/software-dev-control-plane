# Agent Control Model

LinkedIn: https://www.linkedin.com/in/levkantorovich

This project uses a layered control model for AI-assisted development.

## Layers

1. Repository safety: confirm repo root, branch, working tree, and task scope.
2. Task execution: work from a defined task, not vague intent.
3. Skills: load procedural guidance for recurring workflows.
4. Quality gates: run relevant validation and report actual results.
5. Human handoff: explain what changed, why, commands run, results, risks, and next step.

## Control principle

The agent can accelerate execution, but the control plane keeps the work bounded, inspectable, and reviewable.
